SECTION 01

這是什麼？三分鐘快速認識

GameVault 是一個以「單一網頁」運作的 PWA（Progressive Web App）收藏管理工具，專為電玩、動漫周邊的實體收藏設計。核心理念是：資料完全歸你所有——所有收藏記錄實際存放在你自己的 Google 試算表裡，照片存在你自己的 Google Drive，App 本身只是操作介面，不經過任何第三方伺服器儲存你的資料。

🧩三個核心元件

📱 前端網頁（App）

單一 HTML 檔案，負責所有畫面操作，可安裝到手機主畫面像原生 App 一樣使用。

⚙️ Google Apps Script

作為前端與試算表之間的橋樑，處理所有資料讀寫、圖片上傳到 Drive 等後端邏輯。

📊 Google 試算表

真正的資料庫，8 大分類共用一份試算表、分別存放在不同工作表分頁。

📦能管理哪些收藏

8 大分類：遊戲、書籍／攻略、主機、週邊、原聲帶、動漫／美術設定集、公仔／模型、數位下載版，另有獨立的「尋寶清單」追蹤還沒入手、正在比價中的目標商品。

🧭五大主頁籤

畫面最下方固定導覽列，貫穿整個 App：

📷 建檔

新增收藏品，8 種建檔方式可選（詳見第 8 節）。

🗂️ 收藏

瀏覽、搜尋、篩選、編輯已收藏品項（詳見第 10-11 節）。

🎯 尋寶

追蹤比價中的目標商品（詳見第 13 節）。

📊 統計

收藏數量、金額等視覺化統計（詳見第 14 節）。

⚙️ 設定

連線、市場管理、備份等維護功能（詳見第 6 節）。

SECTION 02

事前準備 你需要的帳號與工具

GameVault 是開源自架的工具，要擁有一份「屬於自己」的 GameVault，需要準備以下帳號與資源。如果你是使用別人已經部署好、分享給你的連結，這節可以跳過。

✅必要條件清單

Google 帳號 必要
用來建立 Google 試算表（資料庫）、Google Drive（存放照片）、Google Apps Script（後端程式）。全部都是 Google 帳號內建的免費服務。
GitHub 帳號 必要
存放前端網頁原始碼與部署到 Netlify 的來源。
Netlify 帳號 必要
免費的靜態網頁託管服務，把 GitHub 上的前端網頁部署成可用瀏覽器開啟的網址。
Google Gemini API 金鑰 選用
如果要使用「圖片辨識」「AI 網頁查詢」等 AI 自動填寫功能才需要，純手動建檔可以不用申請。詳見第 4 節。

💰費用

Google 試算表／Drive／Apps Script、GitHub、Netlify 的基本使用額度都是免費的，個人收藏規模的資料量與存取頻率完全在免費額度內。Gemini API 目前有免費額度可供 AI 辨識功能使用。

SECTION 03

部署四步驟 建立你自己的 GameVault

完成第 2 節的帳號準備後，依下列四個步驟即可擁有一份完全屬於自己、資料自主的 GameVault。

1
建立 Google 試算表 ＋ Apps Script 專案

新增一份 Google 試算表作為資料庫，在裡面開啟「擴充功能 → Apps Script」，把 GameVault_AppsScript.gs 的完整內容貼入，儲存。這份程式碼負責回應前端的讀寫請求，並自動建立/管理所有分類需要的工作表分頁。

2
部署為 Web App

在 Apps Script 編輯器右上角「部署 → 新增部署作業」，類型選「網頁應用程式」，執行身分設「我」，存取權設「所有人」。部署完成會得到一組 Web App 網址，這組網址就是前端 App 溝通後端的唯一管道，請妥善保存。

3
取得前端網頁並部署到 Netlify

Fork 或下載 GameVault 的前端原始碼（index.html 等檔案）到自己的 GitHub repo，在 Netlify 建立新站台、連結這個 repo，設定 Publish directory 為存放 index.html 的資料夾。Netlify 會自動幫你產生一個可以直接用瀏覽器開啟的網址。

4
在 App 內完成連線設定

打開 Netlify 部署好的網址，進入「設定」頁，貼上步驟 2 拿到的 Web App 網址並儲存，點「🩺 連線自檢」確認可以正常讀寫。完成後就可以開始建檔使用了。

App 內連線設定欄位
⚠️
Web App 網址一旦確立，之後更新 Apps Script 程式碼時務必用「管理部署作業 → 編輯現有部署」更新版本，而不是每次都新增部署作業，否則網址會改變、需要重新回 App 內修改連線設定。GameVault 專案本身已經設定 GitHub Actions 自動化部署（透過 clasp 工具），一般更新不需要手動操作這一步。
SECTION 04

API 金鑰 AI 辨識引擎

「圖片辨識」「AI 網頁查詢」等自動辨識/填寫功能，背後是呼叫 Google Gemini 的多模態 AI 模型。純手動建檔或使用條碼/編碼查詢資料庫的人可以跳過這節。

🔑 Google Gemini API

前往 Google AI Studio 申請免費的 API 金鑰，用於呼叫 Gemini 模型辨識照片內容、生成商品資訊摘要。目前提供免費額度，個人建檔的使用量通常在額度內。

https://aistudio.google.com/apikey

申請到金鑰後，在 App「設定」頁對應欄位貼上金鑰即可啟用 AI 辨識相關功能（圖片辨識、AI 網頁查詢、條碼+照片、編碼+照片皆需要這組金鑰）。

Google AI Studio 申請 API 金鑰
App 內填入 API 金鑰欄位
ℹ️
API 金鑰屬於機敏資訊，只會存在你自己的瀏覽器本機設定裡，不會上傳到任何第三方伺服器（除了呼叫 Gemini API 本身當下必要的傳輸）。
SECTION 05

八大資料庫 完整說明

GameVault 用 8 個分類（外加尋寶清單）分別管理不同性質的收藏品，每個分類在試算表裡對應獨立的工作表結構。這節詳細說明每個分類的專屬欄位，以及所有分類共用的欄位群組。

共用欄位群組 遊戲 書籍 主機 週邊 原聲帶 動漫／美術設定集 公仔／模型 數位下載版

🧱所有分類共用的欄位群組

不管哪個分類，建檔／編輯表單都會有以下共用欄位群組（各分類差異主要在「版本規格」與規格類群組）：

🖼️ 圖片紀錄
  • 封面圖檔／封底圖檔／側邊圖檔：各限一張，「拍照」直接開相機或「相簿」選取既有照片
  • 自訂圖片：最多 5 張（尋寶清單最多 7 張），每張可自訂名稱標籤，用於燈箱瀏覽的說明文字
🏷️ 識別資訊
  • 資料類別：所屬分類
  • 主要名稱／日文名稱／中文名稱／英文名稱：多語言名稱，估值查詢時依市場語言自動選用對應名稱
  • 系列名稱：所屬作品系列，方便用系列篩選
🔖 編碼條碼
  • 商品編碼：碟片/卡帶/包裝上的產品代碼
  • 條碼 Barcode：EAN／JAN／UPC／ISBN

這兩個欄位也是「關聯商品」判斷是否為同一項商品的重要依據。

🏢 發行廠商
  • 開發商／發行商
  • 發行時間
📦 收藏狀態
  • 收藏狀態：收藏中／已出售／預購中等
  • 特典／附件說明
  • DLC 狀態：已過期／可兌換／已兌換
  • 購入日期／購入價格（含幣別）
  • 入手來源：全新購入／二手購入／贈與
  • 保管位置：自訂收納地點標籤
「收藏狀態」欄位群組展開實際畫面範例
💰 估值資訊
  • 目前估值／估值更新日／估值來源／估值信心／估值備註
  • 估值原幣／原幣估值：保留原始查詢幣別金額不用手動換算
  • 出售價格／出售日期：轉售時記錄實際成交價
🧩 品相完整度

品相／完整度整體評級，另有逐項目評級細項（外盒、說明書、保固卡等各自獨立評等）。

📝 個人記錄

記錄個人心得、收藏緣由等主觀筆記。

🔗 關聯商品

把這個項目跟收藏庫裡其他項目建立關聯（如遊戲本體跟攻略本），可複選。顯示「（未在收藏）」代表關聯目標找不到，通常需要到「設定」頁跑一次「修復工作表標題列」補齊 UUID 後重新連結。

🎵原聲帶

選擇「原聲帶」分類後，需要再選擇 5 個子類型之一：原聲帶、主題曲／單曲、角色歌曲／印象集、廣播劇CD、演唱會音源／其他。不同子類型有各自量身設計的欄位（例如「廣播劇CD」會有聲優陣容欄位，「原聲帶」會有曲目清單欄位），因為欄位差異大，資料庫用不同工作表分開儲存，選對子類型才能對到正確欄位。

🎨動漫／美術設定集

共 8 個子類型：漫畫／單行本、畫冊／插畫集、設定集／公式資料集、原畫集／分鏡集、雜誌／MOOK／同人誌、動畫影集、動畫電影／劇場版、周邊／其他。例如「漫畫／單行本」需要「集數」欄位、「動畫影集」需要「載體格式」欄位，各子類型專屬設計。這個分類另外有依子類型分派的 AI 圖片辨識規格，辨識準確度會依你選的子類型調整判斷標準。

公仔／模型子類型選單

💾數位下載版

共 8 個子類型：下載版遊戲、追加下載內容、電子書（漫畫／單行本）、電子書（畫冊／美術設定）、電子書（攻略／公式書）、電子書（雜誌／MOOK）、數位音源、數位影音。這個分類的商品沒有實體，因此「二手估值市場參考」區塊不會顯示（數位內容通常沒有二手市場可比價）。

數位下載版子類型選單
💡
所有分類（含子類型）的圖片欄位都有「自訂圖片」，張數上限一律湊到「封面＋額外＝8 張」的總數：收藏類別有封面/封底/側邊 3 個固定欄位，自訂圖片上限 5 張；尋寶只有封面 1 個固定欄位，自訂圖片上限 7 張。

連線設定 連線自檢 市場管理 CSV 匯出匯入 其他工具

🔌連線設定

貼上 Google Apps Script Web App 網址並儲存，這是 App 讀寫你自己 Google 試算表資料的唯一管道。首次使用或更換部署網址時才需要修改（詳見第 3 節）。這裡也是填入第 4 節申請的 Gemini API 金鑰的地方。

連線設定區塊

🩺連線自檢 & 修復工作表標題列

🩺 連線自檢
檢測後端連線是否正常、是否具備寫入權限。出現連線異常、資料同步不正常時可以先跑一次確認問題出在哪一環。
🔧 修復工作表標題列
維護試算表結構的重要工具：①自動補齊各分類/子類型工作表缺少的欄位標題（新版本新增欄位後，舊試算表需要跑一次這個才會補上對應欄位）；②自動幫沒有正常 UUID 的舊資料列補上有效的 UUID，這對「關聯商品」連結的穩定性很重要。
連線自檢結果
💡
更新 App 版本後，如果發現新欄位沒有作用、或關聯商品顯示異常，先跑一次「修復工作表標題列」通常就能解決。

🏪二手估值市場管理

「二手估值市場參考」是收藏詳情頁與尋寶詳情頁都會用到的快速比價連結區塊，內建 8 個台日查價網站（露天、Yahoo拍賣、蝦皮、旋轉拍賣、Mercari、ヤフオク、Buyee、日本樂天）。

🏷️ 自訂市場連結

點「＋ 新增市場」加入自己常用的查價網站（最多 12 個），加入後「二手估值市場參考」區塊會改成可左右滑動瀏覽。資料只存本機，隨「匯出／匯入設定」一起攜帶。

①Step 1 · 選擇分類

共 8 個分類，詳細欄位差異見第 5 節。動漫／美術、公仔、原聲帶、數位下載版這 4 個分類選了之後，會再多一個子類型選單。

Step 1 選分類 → Step 2 選區域 → Step 3 選平台

日本／亞洲／美國／其他，影響 AI 辨識與資料庫查詢參考的語言與資料來源。

③Step 3 · 平台選擇

選填提升 AI 辨識與編碼查詢準確率，尤其「商品編碼」建檔方式必須先選好平台才能查得準。

④Step 4 · 建檔方式（8 種）

📼 商品條碼

🔢 商品編碼
輸入碟片/卡帶產品代碼或書籍 ISBN，務必先選平台與區域。
🖼️ 圖片辨識

✏️ 手動建檔
直接進入空白表單自行輸入所有欄位。
📼＋🖼️ 條碼＋照片 推薦
最高準確度，條碼查基本資訊＋照片補齊/校正細節。
🔢＋🖼️ 編碼＋照片
精確比對，適合條碼易失準但知道正確編碼的情況。
🤖 AI 網頁查詢
貼上網路商品資訊文字，AI 解析生成結構化資料。
🔀 自動分流
依輸入格式自動判斷查詢哪個資料來源。

系統判斷分路流程圖

說明系統在不同建檔情境下，如何自動判斷要查詢哪一個資料來源、走哪一條處理路徑。

🔀條碼／編碼查詢分流邏輯

1
判斷輸入格式

系統先檢查輸入的是條碼數字格式（EAN/JAN/UPC/ISBN）還是商品編碼格式（如 CUSA-XXXXX、HAC-XXX）。

2
比對平台／區域

若走「商品編碼」路徑，會依 Step2／Step3 選定的區域與平台，決定用哪套編碼格式規則解析與查詢。

3
查詢對應資料來源

條碼查詢優先比對日版（樂天）商品資料庫；查無資料時提示改用照片辨識或手動輸入。

4
合併結果進入表單

不管走哪條路徑，查詢/辨識到的欄位都會統一帶入同一份表單，供使用者確認與補完。

自動分流判斷結果

SECTION 12

攻略本管理

「書籍／攻略」分類底下，攻略本是使用頻率較高的一種子項目，這節說明攻略本相關的管理方式。

📖攻略本的建檔重點

  • 建檔時分類選「書籍」，版本規格群組會出現對應遊戲的適用平台欄位，方便日後搜尋時能同時用平台篩選攻略本
  • 可以透過「關聯商品」欄位把攻略本連結到對應的遊戲本體，兩者互相參照，在遊戲的詳情頁也能看到「已連結」的攻略本
  • ISBN／條碼查詢對書籍類的資料庫命中率通常較高，建議優先嘗試「商品條碼」建檔方式
攻略本關聯商品範例
💡
攻略本與遊戲本體互相關聯後，之後想找「這款遊戲有沒有攻略本」或「這本攻略本是配哪款遊戲」都可以直接從關聯商品欄位查到，不用另外用搜尋比對名稱。

SECTION 15

換手機／備份與還原

因為資料存在你自己的 Google 試算表，換手機或重新安裝 App 幾乎不會遺失資料，但仍建議了解以下幾種備份還原方式。

資料本身不需要搬移 設定匯出匯入 CSV 匯出匯入

☁️資料本身不需要搬移

因為所有收藏資料存在 Google 試算表、照片存在 Google Drive，換一支手機只要重新開啟 GameVault 的網址（或重新安裝 PWA，見第 7 節），在「設定」頁填入同一組 Web App 連線網址，就能立刻看到所有原本的收藏資料，不需要額外搬移收藏資料本身。

💡
真正需要「備份還原」的是本機端的個人化設定（連線網址、自訂市場清單、市場顯示排序偏好）——這些不是存在試算表裡，換裝置需要重新設定或用下方的「設定匯出匯入」功能快速還原。

⚙️設定匯出／匯入

「設定」頁的匯出／匯入設定功能，備份的是 App 本身的設定（連線網址、自訂市場清單、市場顯示與排序偏好等本機設定），不是收藏資料本身。換手機或重灌 App 時可以用這個快速還原所有個人化設定，不用一個一個重新設定。

設定匯出／匯入

📤CSV 匯出／匯入（收藏資料備份）

雖然資料本身存在試算表不會遺失，但如果想要一份離線、額外的收藏資料備份（例如要遷移到別的系統、或單純想留一份 Excel 檔案），可以用收藏頁「⋯」選單裡的「匯出 CSV」。

匯出

依「分類＋子類型」自動分成多個檔案（避免不同子類型欄位混在同一檔案錯位），採雙標題列格式：

  • 第一列：內部欄位鍵值（供匯入程式辨識）
  • 第二列：中文顯示名稱（方便用 Excel／Google試算表閱讀）
  • 第三列起才是實際資料

圖片欄位會正常匯出（存的是 Google Drive 檔案 ID 短字串，不是圖片檔案本身），匯入後圖片連結能正確保留。

匯入

點「匯入 CSV」會先跳出說明提示，確認後選擇檔案（支援一次選多個）。匯入時會：

  • 自動偵測分類與子類型
  • 用 UUID 判斷是否重複，重複的自動跳過
  • 每一筆新資料都會實際寫回 Google 試算表（不只是存在手機本機），確保重新整理後資料不會消失
  • 支援舊格式（單一標題列）CSV 匯入，但建議都用最新版重新匯出的格式，資料完整性更有保障
CSV 匯出／匯入
⚠️
匯入是逐筆寫入雲端，資料量大時需要一點時間，請耐心等待「匯入完成」提示出現再離開頁面。

新手必讀 注意事項與常見問題

⚠️使用注意事項

  • App 需要網路才能新增／編輯資料（資料要即時寫回 Google 試算表），沒有網路只能瀏覽先前已快取的內容

  • 刪除收藏記錄是無法復原的動作，刪除前請再三確認
  • 建議偶爾使用「連線自檢」確認後端運作正常，尤其是更新 App 版本或很久沒開啟之後

❓常見問題

條碼查詢不到資料怎麼辦？

目前內建條碼資料庫主要涵蓋日版（樂天）商品，台/港/韓/中/美版條碼經常查無資料。建議改用「圖片辨識」拍封面讓 AI 辨識，或直接「手動建檔」輸入。

關聯商品顯示一串英文/數字亂碼而不是商品名稱？

代表該關聯連結使用的識別碼不穩定（目標項目當時還沒有正常 UUID）。到「設定」頁點「🔧 修復工作表標題列」補齊所有項目的 UUID，回到關聯商品欄位把顯示異常的項目移除、用「＋ 關聯商品」重新選一次即可修復。

CSV 匯入後選完檔案沒有任何反應？

請確認選取的是有效的 CSV 檔案且內容完整；若持續沒有反應，可先用「連線自檢」確認後端連線正常，因為匯入需要即時寫回雲端試算表，網路不穩也可能導致延遲。

圖片載入速度變慢？

圖片是直接向 Google Drive 縮圖服務請求，載入速度會受 Drive 服務狀態、你的網路環境、以及單一頁面內圖片數量影響。這部分不是 App 本身能完全控制的環節。

估值市場的搜尋結果不夠精準？

系統會依分類自動帶入平台或分類關鍵字提升準確度（詳見第 6 節市場管理），但仍是關鍵字搜尋，無法保證每次都完全精準，建議把搜尋結果當作比價參考起點，仍需自行核對商品是否相符。

為什麼有些操作沒有網路就不能用？

因為資料實際存在雲端的 Google 試算表，新增/編輯/刪除都需要即時寫回試算表，沒有網路無法完成寫入動作。已經載入過的資料仍可離線瀏覽。
